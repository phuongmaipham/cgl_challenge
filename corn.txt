Corn-specific strategy research with EIA ethanol
Date: 2026-05-01

Objective
---------
Create a corn-only strategy using the same lower-overfit principles as the
soybean work:

- use the provided Cargill signals, including Cargill inventory and Cargill
  crush/activity fields;
- add corn-relevant external ethanol production/stocks from EIA;
- optionally test external FX, macro/risk, relative grain, and Meteostat
  weather/HDD/CDD/GDD signals;
- avoid fitted regression coefficients and avoid Ridge-style coefficient
  fitting;
- log every attempted candidate, including failures.

Important interpretation note
-----------------------------
The provided Cargill crush data is not a tradable corn crush-spread dataset.
For corn it is used as a Cargill physical-processing/activity proxy inside the
provided-data family, while EIA ethanol is the direct corn-demand external
signal.

Data and signal families
------------------------
1. Provided price/trend family
   - mom_20
   - mom_60
   - rev_5
   - curve_spread
   - curve_ratio
   - cot_pm_oi_level

2. Provided physical / Cargill family
   - public_inventory_change, sign flipped so falling inventories are bullish
   - receipts_change, sign flipped
   - cgl_inventory_change, sign flipped
   - crush_surprise
   - crush_utilization
   - curve tightness from curve_spread and curve_ratio

3. EIA ethanol family
   - ethanol_production_change_4w
   - ethanol_prod_to_stocks
   - ethanol_demand_pressure
   - negative ethanol_stocks_change_4w

4. External FX/export family from yfinance
   - USD index pressure
   - BRL pressure
   - CNY pressure

5. External macro/risk family from yfinance
   - crude oil momentum
   - SPY/equity momentum
   - USD trend pressure

6. Meteostat weather family
   - growing-season CDD/HDD/GDD
   - heat stress
   - dryness and dry-CDD interaction
   - planting precipitation/dryness
   - harvest freeze stress

Risk and overfit controls
-------------------------
- No Ridge, no fitted signal coefficients.
- Candidate recipes use fixed economic weights.
- Position sizing uses lagged 60-day corn futures PnL volatility.
- Max position is capped at 0.50 contracts.
- Transaction costs are included in the cost-adjusted results.
- Long-only and long/short mode were both tested, but this is now explicitly
  treated as a research risk because the long/short validation winner failed
  OOS.

What worked
-----------
1. EIA ethanol has real corn relevance and produced the best standalone OOS
   Sharpe. However, standalone ethanol had weak earlier train/validation
   behavior and large full-period drawdown, so it should not be used alone.

2. The best requirement-compliant blended candidate by cost-adjusted OOS
   Sharpe was:

   requirement_given_80_ethanol_10_fx_10

   Formula:

   signal = 0.80 * given_conservative_blend
          + 0.10 * external_ethanol_family
          + 0.10 * external_fx_export_family

   Cost-adjusted OOS:
   - Sharpe: 0.353
   - PnL: 202.606
   - Drawdown: -426.025
   - Full-period Sharpe: 0.311
   - Full-period PnL: 240.165
   - Full-period drawdown: -426.025

3. If drawdown is the priority, the regime hybrid long/short candidate had the
   smallest positive-OOS drawdown:

   regime_hybrid_vol_trend_weight, long_short

   Cost-adjusted OOS:
   - Sharpe: 0.302
   - PnL: 179.817
   - Drawdown: -231.599

   But this candidate had weaker full-period robustness and negative
   validation behavior, so it is not my preferred live recommendation.

What failed / overfit warning
-----------------------------
The strict pre-2018 validation selector chose:

requirement_physical_60_trend_25_ethanol_15, long_short

It looked acceptable in validation:
- zero-cost validation Sharpe: 0.510

But it failed in the untouched 2018-2020 OOS window:
- cost-adjusted OOS Sharpe: -0.487
- cost-adjusted OOS PnL: -438.585
- cost-adjusted OOS drawdown: -936.693

This is exactly why I would not rely on a validation-only winner for corn. Even
without fitted coefficients, selecting among many fixed recipes can still
overfit.

Recommendation
--------------
For a fund-usable corn sleeve, I would use the small-ethanol blended long-only
rule, not the validation-selected long/short rule:

recommended_corn = requirement_given_80_ethanol_10_fx_10, long_only

Pseudo-code:

given_inventory_pressure =
    mean(-public_inventory_change, -receipts_change, -cgl_inventory_change)

cgl_crush_activity =
    mean(crush_surprise, crush_utilization)

given_physical_family =
    mean(given_inventory_pressure, curve_tightness, 0.25 * cgl_crush_activity)

given_price_family =
    mean(mom_20, mom_60, rev_5)

given_trend =
    mean(mom_20, mom_60, curve_spread, cot_pm_oi_level)

given_conservative_blend =
    0.40 * given_physical_family
  + 0.30 * given_trend
  + 0.30 * given_price_family

external_ethanol_family =
    mean(
        ethanol_production_change_4w,
        ethanol_prod_to_stocks,
        ethanol_demand_pressure,
        -ethanol_stocks_change_4w
    )

external_fx_export_family =
    mean(-USD_index_20d_z, -BRL_20d_z, -CNY_20d_z)

raw_signal =
    0.80 * given_conservative_blend
  + 0.10 * external_ethanol_family
  + 0.10 * external_fx_export_family

smoothed_signal = EWMA(raw_signal, halflife=2)
if abs(smoothed_signal) < 0.05:
    smoothed_signal = 0

long_only_signal = max(smoothed_signal, 0)
position = long_only_signal * target_daily_pnl_vol / lagged_60d_corn_pnl_vol
position = clip(position, 0, 0.50)

net_pnl =
    lagged_position * corn_futures_pnl
  - turnover_cost
  - holding_margin_cost

Final judgment
--------------
Corn is weaker than soybeans as a standalone strategy in this dataset. Ethanol
helps the economic story and improves some OOS rows, but it does not create a
high-Sharpe standalone corn strategy. I would include this as a small optional
corn sleeve, not as the main book.

If the fund requires a cleaner risk number, scale the sleeve by allocated
capital. Example:

- OOS DD -426 on 10,000 capital = -4.26%.
- OOS DD -426 on 25,000 capital = -1.70%.
- OOS DD -426 on 50,000 capital = -0.85%.
